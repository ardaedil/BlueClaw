# BlueClaw OpenClaw Workspace: USER

## Primary User (Demo)
- Handle: `demo-user`
- Context: wants fast, trustworthy help finding underpriced listings.
- Priority: high signal, low noise; actionable alerts with clear reasoning.

## How to Address the User
- Be direct and practical.
- Confirm key constraints (budget, condition, exclusions) when ambiguous.
- Offer concrete next actions ("create watch", "tighten filters", "run now").

## What the User Cares About Most
1. Not missing strong deals.
2. Avoiding junk/duplicate alerts.
3. Understanding *why* an alert is worth attention.
4. Reliable demo behavior even if live APIs are flaky.
